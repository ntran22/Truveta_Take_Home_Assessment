# Cascade Regional Health Value-Based Care Dashboard

## Project Overview

This project builds a local Streamlit dashboard for exploring value-based care metrics for the Cascade Regional Health take-home dataset.

The workflow is:

1. Run `truveta_db_setup.py` to create the DuckDB database from the raw CSV files.
2. Run `Truveta Take Home_EDA_Nancy Tran.ipynb` to clean, join, and aggregate the source tables.
3. The notebook outputs `data/combined_table.csv`.
4. Run `Cascade_Regional_Health_Dashboard_App.py` to launch the local Streamlit dashboard using `data/combined_table.csv`.

## Requirements

This project was designed to run locally with Python 3.10+.

Install the required packages:

```bash
pip install pandas numpy duckdb plotly streamlit jupyter itables
```

Core packages used:

- `duckdb` for local SQL querying and database setup
- `pandas` and `numpy` for data cleaning, transformation, and metric calculations
- `plotly` for interactive visualizations
- `streamlit` for the local dashboard app
- `jupyter` for running the EDA notebook
- `itables` for interactive notebook tables, if used in the notebook

## Setup Instructions

### 1. Clone or download the project

Make sure all project files are saved in the same root folder and that the raw CSV files are stored inside the `data/` folder.

The setup script expects the following files:

```text
data/patient.csv
data/encounter.csv
data/billing.csv
data/condition.csv
```

---

### 2. Create the local DuckDB database

Run the database setup script first:

```bash
python truveta_db_setup.py
```

This script creates a local DuckDB database called:

```text
truveta_data.duckdb
```

The script loads the raw CSV files into the following database tables:

- `patient`
- `encounter`
- `billing`
- `condition`

This step must be completed before running the EDA notebook.

---

### 3. Run the EDA notebook

Open and run the notebook:

```bash
jupyter notebook "Truveta Take Home_EDA_Nancy Tran.ipynb"
```

Or, if using JupyterLab:

```bash
jupyter lab
```

The notebook connects to the DuckDB database, performs SQL-based cleaning and aggregation, and generates the cleaned combined dataset:

```text
data/combined_table.csv
```

This file is required by the Streamlit dashboard.

---

### 4. Launch the Streamlit dashboard

After `data/combined_table.csv` has been generated, run:

```bash
streamlit run Cascade_Regional_Health_Dashboard_App.py
```

The dashboard should open in your browser at a local Streamlit URL, usually:

```text
http://localhost:8501
```

## Dashboard Inputs

The Streamlit dashboard reads from:

```text
data/patient.csv
data/combined_table.csv
```

`patient.csv` is used for patient-level demographic fields, and `combined_table.csv` is used as the main backend dataset for dashboard metrics and visualizations.

---

## Key Dashboard Sections

The dashboard includes four main sections:

1. **Executive Overview**
   - Total patients with at least one encounter
   - Total billed amount
   - PMPY estimate
   - PMPY by encounter type
   - Encounter type distribution
   - PMPY trend over time

2. **Population Health**
   - Chronic condition burden
   - Top diagnosis groups
   - Condition prevalence

3. **Cost & Utilization**
   - PMPY by condition group
   - PMPY by census region
   - Monthly billing trends
   - Inpatient vs outpatient utilization

4. **Care Management Opportunities**
   - High-cost patients
   - Super utilizers
   - High-cost and super-utilizer overlap
   - Proxy indicators for potential care gaps and population health opportunities

## Assumptions

- The raw CSV files are available in the `data/` folder before running `truveta_db_setup.py`.
- `truveta_db_setup.py` must be run before the EDA notebook because it creates the local DuckDB database used for SQL queries.
- `Truveta Take Home_EDA_Nancy Tran.ipynb` must be run before launching the Streamlit dashboard because it generates `data/combined_table.csv`.
- `combined_table.csv` is treated as the cleaned backend dataset for the dashboard.
- PMPY is calculated as total billed amount divided by the number of unique patients with at least one encounter and the number of observed years in the filtered dataset.
- Patients included in dashboard metrics are patients with at least one recorded encounter in the selected measurement period.
- Care management flags, including high-cost patients and super utilizers, are proxy measures based on the 90th percentile of total billed amount and encounter volume within the filtered population.
- The care gap metrics shown in the dashboard are not official HEDIS measures. Formal HEDIS care gap reporting would require additional data elements such as enrollment, payer attribution, pharmacy claims, laboratory values, numerator/denominator definitions, and official measure specifications.

## Troubleshooting

### `FileNotFoundError: data/combined_table.csv`

Run the EDA notebook first. The Streamlit dashboard requires `data/combined_table.csv`, which is generated by:

```text
Truveta Take Home_EDA_Nancy Tran.ipynb
```

### `FileNotFoundError: data/patient.csv`

Confirm that the raw CSV files are stored in the `data/` folder.

### DuckDB database not found

Run:

```bash
python truveta_db_setup.py
```

This creates `truveta_data.duckdb`.

### Streamlit command not found

Install Streamlit:

```bash
pip install streamlit
```

Then rerun:

```bash
streamlit run Cascade_Regional_Health_Dashboard_App.py
```
